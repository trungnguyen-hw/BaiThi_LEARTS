## 3. CÁC API ENDPOINTS HIỆN CÓ
- `GET /api/health` - Health check trạng thái CSDL.
- `GET /api/categories` - Lấy danh sách danh mục.
- `GET /api/products` - Tìm kiếm và lọc sản phẩm.
- `GET /api/products/:id` - Chi tiết sản phẩm.
- `POST /api/orders` - Tạo đơn hàng (gồm transaction và giảm trừ stock).
- `POST /api/auth/register` - Đăng ký admin.
- `POST /api/auth/login` - Đăng nhập admin nhận JWT.
- `GET /api/auth/me` - Kiểm tra thông tin JWT Admin hiện tại.

## 4. CẤU TRÚC BẢNG VÀ KIỂU DỮ LIỆU ID HIỆN TẠI
- **`categories`**: `id` dùng kiểu dữ liệu **TEXT** (chuỗi chữ như `kitchen`, `toys`, `decor`, `accessories`).
- **`products`**: `id` dùng kiểu dữ liệu **TEXT** (chuỗi chữ như `boho-beard-mug`, `motorized-tricycle`).
- **`admins`**: `id` dùng kiểu tự tăng (MySQL: `INT AUTO_INCREMENT` $\rightarrow$ PostgreSQL: `BIGINT GENERATED BY DEFAULT AS IDENTITY`).
- **`orders`**: `id` dùng kiểu tự tăng (MySQL: `INT AUTO_INCREMENT` $\rightarrow$ PostgreSQL: `BIGINT GENERATED BY DEFAULT AS IDENTITY`).
- **`order_items`**: `id` dùng kiểu tự tăng (MySQL: `INT AUTO_INCREMENT` $\rightarrow$ PostgreSQL: `BIGINT GENERATED BY DEFAULT AS IDENTITY`).

## 5. LIÊN KẾT GIỮA FRONTEND VÀ BACKEND (CONTRACTS)
- Frontend giao tiếp thông qua Axios với baseURL `/api` (Vite Proxy chuyển hướng tới `http://localhost:5000`).
- Các trường số như `price`, `oldPrice`, `rating` được client parse thành Number.
- Các trường Boolean như `featured`, `hot`, `isNew`, `sale` phải trả về đúng `true`/`false`.
- Badge object cấu hình dựa trên thuộc tính khuyến mãi (`sale`), nổi bật (`hot`), và mới (`is_new`).

## 6. RỦI RO LÀM HỎNG FRONTEND VÀ BIỆN PHÁP KHẮC PHỤC
1. **Numeric dạng String từ `pg`**: Thư viện `pg` mặc định trả về kiểu NUMERIC dưới dạng chuỗi (String) để tránh mất độ chính xác.
   - *Biện pháp*: Parse giá trị sang `Number(value)` trước khi trả về API.
2. **Boolean dạng 0 / 1**: MySQL trả về `TINYINT(1)` thành `0` hoặc `1`. Frontend có thể kiểm tra dạng `row.featured === true`.
   - *Biện pháp*: Chuyển đổi tường minh sang boolean: `!!row.featured`.
3. **Product ID kiểu chuỗi**: Nếu sử dụng ID tự tăng kiểu số cho sản phẩm, các liên kết route chi tiết (`/product/boho-beard-mug`) sẽ lỗi.
   - *Biện pháp*: Giữ nguyên kiểu dữ liệu ID của `categories` và `products` dưới dạng `TEXT`/`VARCHAR` và chèn đúng ID chuỗi gốc.
4. **Transaction Rollback không giải phóng client**: Trong PostgreSQL, nếu câu lệnh trong transaction lỗi, client của pool phải được rollback và release về pool, nếu không sẽ làm đơ (leak) kết nối.
   - *Biện pháp*: Đảm bảo dùng khối `try..catch..finally` giải phóng client qua `client.release()`.
